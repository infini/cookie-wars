# 쿠키전쟁 게임 상세 명세

## 제품 정보

- 이름: 쿠키전쟁 (Cookie Wars)
- Android 패키지: `com.cookiewars.game`
- 플랫폼: Android 전용 APK
- 화면: 스마트폰 세로 고정
- 플레이: 한국어 싱글 플레이
- 대상: 어린이. 큰 터치 영역, 짧은 문장, 아이콘 중심, 밝은 쿠키 색상을 사용

## 핵심 흐름

```text
게임 화면에서 쿠키 클릭
→ 쿠키 획득
→ 쿠키 업그레이드 또는 원반·쿠키봇 구매
→ 난이도 선택
→ 전투 화면에서 방어 전투
→ 승리 횟수 기록 및 해당 전투 최초 클리어 거대 원반·전투 훈장 보상 판정
→ 로비로 복귀하거나 다음 전투를 즉시 시작
```

게임 화면에는 적, 원반, 쿠키봇, 전투가 나타나지 않습니다. 전투 화면에서만 적, 봇 자동 공격, 원반, 승패가 작동합니다.

## 공통 내비게이션

모든 화면 하단에는 게임, 전투, 강화, 정보의 대분류 4개만 같은 순서로 표시합니다. 게임과 전투는 자식 화면이 하나이므로 소메뉴를 표시하지 않고 한 번의 대분류 탭으로 즉시 진입합니다. 따라서 전투 화면의 전장 높이는 소메뉴 때문에 줄어들지 않습니다.

강화 대분류는 쿠키 강화, 원반 강화, 쿠키봇 강화의 소메뉴를 표시하고 정보 대분류는 쿠키·훈장, 난이도, 몬스터 소메뉴를 표시합니다. 대분류를 처음 누르면 각각 쿠키 강화와 쿠키·훈장으로 진입하며, 같은 앱 실행 세션에서는 그 대분류에서 마지막으로 본 소화면을 다시 엽니다. 새로운 몬스터가 있으면 정보 대분류와 몬스터 소메뉴 모두에 NEW 점을 표시합니다. 메뉴 순서·한국어 라벨·아이콘·기본 화면·배지 연결은 `src/navigation/navigation.json`에서 관리합니다.

## 게임·쿠키 성장

상단 통계 아래의 가운데 영역 어디든 손가락이 닿으면 쿠키가 정확히 한 번 증가하고 클릭 애니메이션, `+수량` 텍스트, 효과음, 약한 진동이 한 번 발생합니다. 넓은 hit slop과 이동 허용 범위를 사용하고 터치 종료를 기다리지 않아 빠른 연속 입력도 잘 인식합니다. 일반 클릭은 현재 `clickPower`만큼 지급합니다. 메인 상단에는 `쿠키전쟁`과 현재 진화 쿠키 이름을 한 줄로 표시하고, 보유 쿠키·진화 레벨 아래에는 다음 쿠키까지 남은 강화 수와 진행 막대를 둡니다. 중앙에는 확대된 쿠키를 표시하며 안내 문구·다음 쿠키 이름·별도 전투 이동 버튼은 두지 않습니다. 하단은 전투 훈장, 자동 생산, 클릭 시 획득 순서이고, 일반/슈퍼 크리티컬과 마그마/전기 조각은 전용 아이콘 두 줄로 묶습니다. 전투는 공통 하단 `전투` 메뉴로 이동합니다. 쿠키 크리티컬은 기본 1%·10배, 슈퍼 크리티컬은 기본 0.1%·100배이며 배타 구간으로 판정합니다. 일반 크리티컬은 화면을 가리는 이펙트 없이 노란 획득 숫자와 전용 음향만 표시합니다. 슈퍼는 충전 마름모, 3회 확장 충격파, 순차 참격, 8갈래 번개, 회전 엠블럼과 쿠키 무대 흔들림을 조합하고 시네마틱 충격과 전기 폭발 `콰광`을 겹칩니다. 슈퍼·마그마·전기는 서로 겹쳐 합성하고 같은 종류가 다시 나오면 이전 같은 종류만 새 연출로 교체합니다. 수명·음향 레이어·화면 밀도는 기본 확률의 역순인 `전기 > 슈퍼 > 마그마 > 일반`으로 강해집니다.

쿠키 클릭마다 크리티컬과 분리된 난수로 마그마 조각 0.2%, 전기 조각 0.05%를 판정합니다. 발견한 조각은 쿠키 옆으로 부스러기를 흩뿌리며 좌우에 번갈아 튀어나오고 등장 완료 후 정확히 5초 동안만 누를 수 있습니다. 수집 조각은 초기 112px의 정확히 50%인 56px이지만 터치 여백은 40px로 넓혀 누르기 쉽게 유지합니다. 마그마는 기본 클릭 힘의 50배와 화산 분화구에서 직접 솟는 프로젝트 전용 수직 화염, 4회 맥동·좌우 흔들림, 3중 타원 충격파·16개 불씨와 두 겹 화염 주문음을 제공합니다. 전기는 200배와 서로 다른 외부 CC0 원본 낙뢰 11개의 시차 점멸, 3중 마름모 충격파·18개 방사 파편·회전 코어, 세 번의 강한 근거리 천둥음을 제공합니다. 두 연출은 중앙 플레이 공간 전체를 불투명하게 덮지 않습니다. 별도 `획득` 문구는 띄우지 않고 실제 증가량과 배수만 표시합니다. 조각 터치는 기본 클릭으로 전파하지 않습니다. 자동 생산은 앱이 종료되어도 마지막 저장 시각부터 다음 실행 시각까지 완료된 주기를 저장 당시 강화·진화 배율로 누락 없이 정산하며 임의 시간 제한은 없습니다.

네 희귀 보상은 서로 독립한 연속 실패 횟수를 갖습니다. 현재 확률이 `p`이면 `ceil(1/p)`번째 시도까지는 반드시 발동합니다. 예를 들어 10% 일반 크리티컬은 9회 연속 실패하면 10번째에 확정입니다. 슈퍼 발동은 일반·슈퍼 실패 횟수를 모두 초기화하고, 두 조각이 같은 클릭에서 확정이면 전기를 먼저 주며 마그마 보정은 다음 클릭까지 유지합니다. 실패 횟수는 저장되어 앱 종료로 리셋되지 않습니다.

강화 화면에는 클릭당 획득, 쿠키 크리티컬, 쿠키 슈퍼 크리티컬, 마그마 조각 발견, 전기 조각 발견, 자동 생산, 쿠키 성 체력을 표시합니다. 각 카드에는 현재 단계·현재 능력·다음 능력·가격을 표시하고, 바로 강화할 수 있는 카드를 위에 두며 그 안에서는 다음 비용이 저렴한 순서로 정렬합니다. 화면에 보이는 7종은 모두 최고 레벨 없이 계속됩니다. 일반·슈퍼 크리티컬·마그마·전기는 종류마다 레벨당 발견 확률이 자기 기본 확률의 25%, 획득 배수는 자기 기본 배수의 10%만큼 같은 비율로 증가합니다. 실제 확률은 각각 50%·10%·5%·1.5%에서 멈추지만 배수와 강화 단계는 계속 증가합니다. 저장된 현재 레벨에 같은 공식을 적용하므로 기존 진행도도 새 조각 배수까지 즉시 소급 보정됩니다. 화면에 보이는 7종의 현재 단계가 모두 쿠키 진화 레벨에 포함되므로, 이미 올린 슈퍼·두 조각 강화도 다음 쿠키까지 남은 강화 횟수를 즉시 줄입니다. 숨겨진 이전 쿠키 크기만 진화 합계에서 제외하며 저장 이전과 최대 렌더 크기 계산을 위한 데이터로만 남겨 새로 구매할 수 없게 합니다. 메인 쿠키 렌더 크기는 이 데이터 행의 최고 비율과 280px 상한으로 계산한 최대 크기에 고정합니다.

클릭 힘은 Lv.1 기본 50이고 강화마다 +50입니다. 명시된 Lv.8은 400이며 이후 증가량도 `cookie-upgrade-rules.json`에서 +50으로 관리합니다.

쿠키에는 별도 경험치가 없습니다. 강화 화면에 보이는 클릭 힘·일반 크리티컬·슈퍼 크리티컬·마그마·전기·자동 생산·쿠키 성 체력의 현재 레벨 합계에 기존 저장의 `legacyCookieEvolutionBonusLevels`를 더한 값이 진화 레벨입니다. 일곱 강화가 모두 Lv.1인 새 게임의 기본 합계는 7입니다. 새 게임의 이전 보너스는 0이며 v7 이하 사용자는 예전 쿠키 크기의 `Lv - 1`을 한 번 이전받습니다. 기존 진화 조건은 3, 9, 15, …, 297이고 확장 조건은 375, 387, …, 723입니다. 이전 이후에는 쿠키 크기 레벨을 진화 계산에 사용하지 않습니다.

진화 레벨이 `cookies.json`의 조건에 도달하면 자동으로 더 좋은 종류로 바뀝니다. 현재 80종이며 높은 쿠키일수록 클릭 획득·자동 생산·쿠키 성 HP에 더 큰 배율을 줍니다. 기존 쿠키왕국 심장(297·×62.06) 뒤에는 수정 용의 알(375·×68.27)부터 12레벨 간격으로 30종이 이어져 쿠키 우주핵(723·×1083.16)에 도달합니다. 확장 단계의 배율과 간격은 `cookie-expansion.json`에서 관리합니다. 괄호는 필요 진화 레벨과 세 능력의 공통 배율입니다. 메인 화면은 현재 진화 레벨·현재 쿠키 이름과 다음 진화까지 남은 강화 수·진행 막대를 간결하게 표시합니다. 다음 쿠키 이름·정확한 필요 레벨과 전체 진화 순서는 `정보 > 쿠키·훈장`에서 확인합니다.

전투에서 새 스테이지를 진행해 이길 때마다 전투 훈장 1개를 얻습니다. 훈장 1개는 클릭 힘·자동 생산·쿠키 성 체력을 각각 영구 1% 높입니다. 게임 화면에는 현재 훈장과 성장률을 간단히 표시하고 `정보 > 쿠키·훈장`에는 보유량, 세 능력의 현재 퍼센트·배율, 다음 훈장 획득 전투와 `강화 × 쿠키 진화 × 훈장 배율` 계산식을 표시합니다. 예를 들어 300개면 `1 + 300/100 = ×4.00`입니다. 크리티컬·슈퍼 크리티컬·마그마·전기 조각도 이 최종 클릭 힘을 사용합니다. 훈장은 쿠키 진화 레벨이나 쿠키 종류의 해금 조건에는 포함하지 않습니다. 지급량과 능력별 증가율은 `battle-rewards.json`에서 관리합니다.

## 전투

아군은 화면 아래, 거대 보스는 위쪽 중앙에서 시작합니다. 모든 아군 개체 이름은 파란색, 보스 이름은 빨간색으로 객체 바로 위에 표시합니다. 전투마다 선택 난이도에 연결된 고유 보스 한 마리만 등장하며 중앙의 열린 공간을 가로질러 쿠키 성을 향해 천천히 접근합니다. 다중 경로, 적 행렬, 순차 출현은 사용하지 않습니다.

HUD의 속도 버튼은 X1 → X2 → X3 순서로 순환합니다. 선택한 속도는 저장되어 다음 실행과 다음 전투에도 유지됩니다. 속도를 높이면 애니메이션만 빨리 돌리는 것이 아니라 50ms 주기에서 측정·제한한 경과 시간의 전투 단계를 같은 화면 갱신 안에서 각각 1·2·3회 순차 실행하므로 이동, 쿨타임, 발사, 충돌, 피해와 승패가 모두 같은 배율로 진행됩니다.

OS 화면 절전 방지는 전투 상태가 `active`인 동안에만 켭니다. 전투 준비 화면, 승리·패배 결과 화면, 로비와 다른 메뉴에서는 Wake Lock을 해제해 스마트폰의 원래 자동 절전 시간을 그대로 따릅니다. 전투 화면을 벗어나는 경우에도 cleanup에서 즉시 해제합니다.

전투 배경은 길과 갈림길 없이 중앙 70%를 넓게 비우되, 색만 바꾼 한 장의 맵을 사용하지 않습니다. 기존 15개 전장 뒤에 수정 광산, 버섯 숲, 증기 주조소, 심해 산호 도시, 운석 분화구, 장난감 공방, 마법 도서관, 그림자 극장, 연금 온실, 성운 설계실, 거울 미궁, 프랙탈 협곡, 역중력 도시, 영겁 시계, 다중우주 균열을 더해 총 30개 고유 전장을 사용합니다. 지형·랜드마크·가장자리 구조 자체가 다르고 같은 난이도의 1~20번째 전투는 같은 테마를 유지합니다. 각 배경의 중앙은 유닛보다 낮은 대비로 유지합니다. 상단에는 전투 정보와 화면 너비 88%의 긴 보스 HP 바를 표시해 현재 피해량을 즉시 확인합니다. 보스는 최대 112 크기로 표시하고 5종 봇은 성 양옆의 좁은 슬롯에 배치합니다. 유닛은 맵과 같은 높은 시점, 접촉 그림자와 Y 좌표 기반 겹침 순서를 사용합니다.

몬스터 데이터와 도감에는 기존 일반 적 4종과 난이도별 고유 보스 30종을 정의합니다. 각 보스는 갑옷, 얼굴, 몸체 재료, 거대 전쟁망치 실루엣이 다르고 `enemy-waves.json`의 난이도별 결투 행을 통해 연결됩니다. `difficulties.json`의 모든 난이도는 적 수를 1로 고정하고, 30종 보스의 기준 전투 능력치는 같아서 외형 변화가 밸런스를 바꾸지 않습니다.

쿠키봇은 구매한 수량만큼 장착 원반의 피해가 합산되며 공격 반경 안으로 들어온 보스에게 쿠키 원반을 자동으로 던집니다. 너무 먼 보스는 공격하지 않습니다. 전투 화면의 5종 쿠키봇은 쿠키를 들고 있으며 슬롯에 붙은 정지 그림이 아니라 보스를 향해 전진·좌우 순찰하는 실제 위치 이동을 합니다. 종류마다 달리기 3프레임과 투척 준비·놓기·복귀 3프레임을 사용하고, 원반 발사점에 테이블의 손 높이 offset을 적용해 실제 손에서 원반이 빠져나가는 동작을 보여 줍니다. 자동 공격은 항상 작동하며 끌 수 없습니다.

### 쿠키봇 공격과 자동 전투

쿠키봇은 자동 전투 설정과 관계없이 종류별 공격 간격에 맞춰 장착 원반을 자동으로 발사합니다. 같은 종류의 봇을 여러 대 보유하면 한 발의 피해가 수량만큼 합산됩니다.

자동 전투가 OFF이면 사용자가 유닛과 빈 공간을 포함한 전투장 어디든 눌렀을 때 쿨타임이 끝났고 공격 반경 안에 적이 있을 경우에만 성이 한 번 발사합니다. 자동 전투를 ON으로 바꾸면 성도 같은 쿨타임·사거리 규칙으로 자동 발사하며, 승리 보상 저장이 끝난 뒤 다음 전투를 자동 시작합니다. 20번째 전투 승리로 난이도가 바뀌면 새 난이도 1번째 전투로 이어지고 마지막 `black sun extreme god` 20번째 전투에서는 연속 진행을 끝냅니다. 패배하면 자동 재도전하지 않으며 결과 화면에 머뭅니다. 설정은 저장되어 다음 전투와 재실행 후에도 유지됩니다. 성 원반의 피해는 장착 원반 기본 피해의 2배이고 크기는 장착 원반의 기존 크기 100%를 유지합니다. 이전 원반이 화면에 남아 있어도 쿨타임만 끝나면 새 원반을 발사할 수 있습니다.

쿠키봇 자동 원반은 성 중앙이 아니라 화면에 표시된 각 쿠키봇 위치에서 출발합니다. 봇 원반은 성 원반과 쉽게 구분되도록 `battle-rules.json`의 배율에 따라 더 작게 표시하며 현재 크기는 성 원반의 55%입니다.

### 거대 원반

각 전투 번호를 처음 클리어하면 거대 원반을 1개 얻습니다. 거대 원반은 전투 화면의 보유량 버튼으로만 사용하며 발사할 때마다 1개가 영구 저장 수량에서 차감됩니다. 보유량이 0이거나 살아 있는 보스가 없으면 사용할 수 없습니다.

거대 원반 피해는 현재 군단에서 가장 강한 쿠키봇이 날리는 일반 원반 한 발의 정확히 30배입니다. 봇이 없는 예외 상태에서는 장착 원반 기본 피해의 30배를 사용합니다. 이 수치가 발사체의 `damage`에 저장되고 명중 시 보스 HP에서 실제로 차감됩니다. 크기는 휴대폰 화면 너비의 34%로 보스만큼 크게 보이고, 일반 원반보다 천천히 회전하며 날아갑니다. 발사 중에는 금색·푸른색 두 겹 오라, 발광과 맥동 효과, `거대 원반!` 안내가 함께 나타납니다. 피해·속도·크기·이펙트의 모든 게임 수치는 `giant-disc.json`에서 관리합니다.

전투 시작에는 장착할 원반과 쿠키봇이 각각 하나 이상 필요합니다. 쿠키 성은 성벽과 세 개의 탑이 보이는 별도 3D 성 그래픽이며 중앙에 현재 진화 쿠키 문양을 표시합니다. 업그레이드와 현재 쿠키 배율로 증가하는 HP가 있고 적 원반과 근접 공격에 피해를 받습니다. 성 HP가 0이면 아군 패배, 모든 적 HP가 0이면 아군 승리입니다. 성과 적 체력 게이지에는 진한 외곽선이 있으며 남은 체력이 많을 때 초록, 줄어들수록 노랑·주황을 거쳐 빨강으로 연속 변화합니다.

## 원반·쿠키봇 상점

원반 화면에는 원반 5종만, 쿠키봇 화면에는 쿠키봇 5종만 표시합니다. 초코칩·우유·민트·무지개·왕관 원반과 초코·밀크·민트·무지개·로열 쿠키봇으로 초급부터 최상급까지 구성합니다. 모두 비교적 낮은 초기 가격으로 시작하며 봇의 다음 구매 가격은 테이블 배율로 완만하게 증가합니다. 원반은 종류별로 영구 소유하고 하나를 장착합니다.

원반은 상위 종류로 갈수록 Lv.1 공격력이 3 → 10 → 24 → 50 → 130으로 크게 오르고 크기·속도도 증가하며 쿨타임은 짧아집니다. 쿠키봇 피해 배율도 ×1 → ×2.2 → ×4.6 → ×9 → ×25, 발사 간격은 1.8초 → 0.35초로 벌어집니다. 화면 카드에서 능력치를 함께 비교하며 봇 색상, 가격과 모든 능력치는 `bots.json`에서 수정합니다.

모든 원반에는 최고 레벨이 없습니다. 1~5레벨은 종류별 개별 값이고 이후에는 공통 무한 강화 테이블로 공격력·크기·속도를 계속 높입니다. 쿨타임은 안전 하한까지만 감소하지만 다른 능력과 강화 레벨은 계속 증가합니다.

## 승리, 보상, 난이도

난이도는 드롭다운 하나에서 선택합니다. easy만 처음 열려 있고 다음 순서로 진행합니다.

easy → normal → hard → harder → insane → easy demon → medium demon → hard demon → insane demon → extreme demon → easy god → medium god → hard god → insane god → extreme god

→ blood moon easy → blood moon normal → blood moon hard → blood moon harder → blood moon insane → blood moon easy demon → blood moon medium demon → blood moon hard demon → blood moon insane demon → blood moon extreme demon → blood moon easy god → blood moon medium god → blood moon hard god → blood moon insane god → blood moon extreme god
→ black sun easy → black sun normal → black sun hard → black sun harder → black sun insane → black sun easy demon → black sun medium demon → black sun hard demon → black sun insane demon → black sun extreme demon → black sun easy god → black sun medium god → black sun hard god → black sun insane god → black sun extreme god

각 난이도에서 누적 20승을 해야 바로 다음 난이도가 열립니다. 19승까지는 열리지 않습니다. 열린 전투는 계속 재도전할 수 있습니다.

한 난이도 안에서도 1번째부터 20번째까지 보스 강도가 달라집니다. 승리할 때마다 다음 보스의 HP 배율은 easy 기준 0.008, 공격 배율은 easy 기준 0.005, 이동 속도는 easy 기준 0.012씩 더해집니다. 적 원반 레벨은 등급 안에서는 유지하고 다음 난이도에서 한 단계 상승합니다. 기존 15개 경계는 이전 20번째보다 한 단계 높고, 뒤의 Blood Moon·Black Sun 30개는 각각 직전 난이도 20번째보다 보스 HP·공격 배율과 적 원반 피해가 20% 높게 시작합니다. 이동속도와 적 원반 크기·속도·쿨타임은 플레이 불가능한 값으로 발산하지 않도록 `difficulty-expansion.json`의 상한·하한을 적용합니다. 보스 수는 모든 단계에서 한 마리로 고정됩니다. 패배하면 같은 전투 단계에 다시 도전하며 20승 이후에는 20번째 강도를 유지합니다. 스테이지 증가식은 `battle-stage-rules.json`, 두 확장 시리즈의 경계식은 `difficulty-expansion.json`에서 관리합니다.

단일 보스가 성장한 쿠키봇 군단의 첫 자동 원반에 즉사하지 않도록 전력 하한을 함께 사용합니다. 현재 장착 원반과 종류별 봇 수량으로 실제 자동 DPS를 계산해 난이도에 맞는 최소 생존 시간만큼 HP를 확보하고, 가장 강한 자동 원반도 최소 6번은 맞아야 쓰러지게 합니다. 초반에는 기존 테이블 HP가 더 높아 그대로 적용되므로 업그레이드 효과가 유지되고, 봇 수십 대를 보유한 고성장 저장에서만 자동 보정이 우선됩니다. 성의 수동 원반과 거대 원반은 전력 보정에서 제외합니다.

거대 원반과 전투 훈장은 각 난이도의 1~20 전투마다 처음 진행해 이길 때 각각 1개를 지급합니다. 다음 전투로 진행해 처음 이기면 두 보상을 다시 받습니다. 이미 완료한 전투를 재도전할 때는 둘 다 추가 지급하지 않으며, 20승 뒤 반복 플레이는 20번 전투 재도전으로 처리합니다. 전투 승리로 쿠키를 직접 지급하지 않고 전투 훈장이 클릭 힘·자동 생산·쿠키 성 체력을 영구 강화합니다. 거대 원반 보상 수령 키, 난이도별 승리 진행도와 누적 전투 훈장은 별도로 저장합니다.

v9 이전 저장은 이미 진행한 난이도별 승리 수를 먼저 0~20 범위로 정규화한 뒤 모두 합산하고, 완료 스테이지 1개당 훈장 1개를 한 번 소급 지급합니다. 예를 들어 easy 20승과 normal 7승을 보유한 저장은 처음 v9로 열 때 전투 훈장 27개와 세 능력 +27%를 얻습니다. 이후에는 저장된 훈장 수를 사용하므로 앱을 다시 실행해도 같은 보상이 중복되지 않습니다.

v13에서 기존 15개를 모두 완료한 저장은 v14 최초 실행 시 `blood moon easy`를 자동 해금·선택합니다. v14에서 30개를 모두 완료한 저장은 v15 최초 실행 시 `black sun easy`를 자동 해금·선택합니다. 이미 받은 스테이지·훈장·거대 원반 기록은 그대로 보존하고 신규 난이도 진행도만 0에서 시작합니다.

승리와 패배 결과 창에는 두 버튼을 표시합니다. 왼쪽 `로비 이동`은 메인 쿠키 화면으로 돌아갑니다. 자동 전투 OFF에서 오른쪽은 승리 시 `다음 전투`, 패배 시 `다시 전투`입니다. 자동 전투 ON으로 승리하면 다음 전투 준비 문구와 `자동 전투 끄기`를 표시하고 설정된 짧은 지연 뒤 자동 진행합니다. 패배했을 때는 자동 설정이 켜져 있어도 `다시 전투`를 사용자가 눌러야 합니다.

난이도 차이는 고유 보스·맵 외형과 보스의 HP·공격 배율, 이동 속도, 적 원반 레벨입니다. 난이도가 올라도 적 수는 한 마리로 유지됩니다. 보스는 근접 지점에 도달하기 전부터 공격 반경 안에서 빨간 원반을 발사하고, 성 앞에 도달하면 별도 쿨타임의 근접 공격도 함께 반복합니다. 플레이어 원반은 회피하지 않습니다. 기본 보스 원반·근접 공격 피해 배율은 2, 공격 간격 배율은 0.5여서 공격 속도는 2배입니다. 여기에 최신 전체 난이도 배율 1.2가 HP와 최종 피해에 적용됩니다. 이동 속도는 난이도·스테이지 결과의 80%로 적용해 20% 느립니다. 난이도 화면에는 보스 수 1마리와 선택 난이도의 결투 이름을 표시합니다.

보스가 원거리 또는 근접 공격을 하기 600ms 전에 점선 붉은 오라와 크기 변화로 예고하고, 공격 순간에 성 방향으로 짧게 돌진합니다. 공격 예고 오라는 낮은 투명도의 점선으로 그려 명중 이펙트와 혼동하지 않습니다. 보스가 맞으면 중심 한 점의 붉은 원 대신 서로 다른 다섯 좌표에서 금색·흰색·파란색 섬광과 방사형 파편이 시간차로 터집니다. 성 수동 원반과 거대 원반의 강한 타격에는 전장의 빈 공간을 활용한 얇은 충격파가 추가되고, 맞은 개체만 짧게 흔들립니다. 적 원반에는 빨간 궤적이 남아 아군 원반과 즉시 구분됩니다. 보스 HP가 50% 이하가 되면 `보스 분노!` 페이즈로 전환하여 오라·공격 속도·피해가 추가로 강화됩니다. 전투 연출 분석 근거와 적용 범위는 `DEFENSE_GAME_RESEARCH.md`에 문서화했습니다.

난이도별 보스는 전장에 등장할 때부터 자신의 테마와 맞는 거대 쿠키 전쟁망치를 손에 들고 있으며 이동·대기 중에도 망치가 사라지지 않습니다. 각 보스는 실제 걷기 3프레임과 망치 준비·충돌·복귀 3프레임을 가져 30종 합계 180개 투명 WebP 프레임을 사용합니다. 걷는 동안 이동 거리로 프레임을 선택합니다. 강공격 주기는 보스가 공격 사거리에 들어온 시점부터 5초입니다. 다음 `special`을 앞두고 적 원반 비행 예상 시간과 900ms 준비 시간을 합친 구간에는 일반 원거리 공격 채널을 예약하며, 주기가 끝나면 일반 원반 쿨다운을 기다리지 않고 `special` 한 발을 우선 발사해 준비 동작과 일반탄이 겹치거나 강공격이 지연되지 않게 합니다. 실제 강공격이 실행될 때만 준비·충돌·복귀 프레임을 재생하고, 일반 원거리 공격이나 근접 공격마다 강한 망치 동작을 재생하지 않습니다. 충돌 순간에는 보스 발밑의 지면 균열, 타원형 먼지 충격파, 짧은 전장 흔들림·플래시와 강한 타격 효과음이 함께 나타납니다. 추상적인 초승달이나 부메랑형 효과는 사용하지 않습니다. 강공격 원반도 일반 적 원반보다 크고 밝게 표시합니다. 별도 피해 배율은 만들지 않아 현재 전투 밸런스는 바꾸지 않습니다. 강공격 주기·예고·지면 충격·발사체·플래시는 `boss-special-attack.json`, 보스별 프레임과 걷기·충돌·복귀 시간은 `boss-animation.json`에서 관리합니다.

전투에서 처음 만난 몬스터는 도감에서 해금되고 `NEW`를 표시합니다. 도감 화면을 확인하면 NEW 상태를 해제합니다. 아직 만나지 않은 몬스터는 이름을 `?????`로 표시합니다. 해금된 항목은 이름, 체력, 공격력, 설명을 보여줍니다.

## 설정과 피드백

사운드와 진동은 각각 켜고 끌 수 있습니다. 효과음은 1~5의 다섯 단계이며 단계 버튼을 누르면 실제 일반 쿠키 클릭음을 해당 볼륨으로 미리 듣습니다. 일반 클릭은 Freesound CC0 `Crunch` 녹음을 작은 속도 차이의 세 보이스로 번갈아 재생합니다. 일반 크리티컬은 Mixkit 충격음과 반짝임, 마그마는 `Big fire spell burning`을 180ms 간격으로 두 겹, 슈퍼는 시네마틱 충격과 `Heavy electric shockwave impact`, 전기는 저급한 지지직 없이 `Strong close thunder explosion`을 최대 상대 음량으로 220ms 간격 세 번 재생합니다. 음향 레이어 강도는 `전기 > 슈퍼 > 마그마 > 일반`이고 테이블 검증이 역전을 거부합니다. 같은 종류만 이전 레이어와 예약 재생을 정지·되감고 서로 다른 종류는 함께 합성합니다. 수치는 `cookie-feedback.json`과 `cookie-fragments.json`, 전역 5단계 음량은 `audio-settings.json`에서 관리합니다. 화면 전환·사운드 OFF는 모든 쿠키 레이어와 예약음을 즉시 정지합니다. 전투는 CC0 오케스트라·액션음을 사용하고 승패 판정·로비 이동 즉시 모두 정지하며, 박수·팡파르는 재생하지 않습니다.

## 저장 항목

- 현재·누적 쿠키
- 모든 쿠키 업그레이드 단계
- 이전 쿠키 크기 진행도를 한 번 옮긴 진화 보너스 레벨
- 소유 원반 목록, 종류별 원반 단계, 장착 원반
- 종류별 쿠키봇 수
- 선택 난이도, 최고 해금 난이도, 난이도별 승리 수
- 난이도·전투 번호별 최초 클리어 보상 수령 여부
- 보유 거대 원반 개수
- 보유 전투 훈장 개수
- 발견 몬스터와 NEW 상태
- 사운드 ON/OFF, 5단계 볼륨, 진동 ON/OFF
- X1·X2·X3 전투 속도
- 자동 전투 ON/OFF
- 일반·슈퍼 크리티컬과 마그마·전기 조각의 연속 실패 횟수

진행은 AsyncStorage에 자동 저장하며 앱을 다시 실행하면 복구합니다.

현재 저장 스키마는 `saveVersion` 15입니다. 이전 저장에 두 크리티컬·두 조각 발견 단계가 없으면 기본 Lv.1, 전투 속도가 없거나 허용값이 아니면 X1, 자동 전투가 없으면 OFF, 4종 실패 횟수가 없으면 0으로 복구합니다. 기존 15개 완료 저장은 최초 Blood Moon, 기존 30개 완료 저장은 최초 Black Sun 난이도로 데이터 기반 마이그레이션에서 한 번 연결합니다.
