# 작업 진행 기록

## 2026-07-19 · 무한 쿠키 조각 성장·합성 피드백·강한 낙뢰음

### 요청

- 쿠키 클릭 때 0.2% 마그마·0.05% 전기 조각을 옆으로 튀어나오게 하고 5초 안에 누르면 Lv.1 기준 클릭 힘 50배·200배 지급
- 조각 강화도 크리티컬처럼 발견 확률과 획득 배수가 함께 무한 성장하고, 네 희귀 보상의 성장 비율을 동일하게 구성
- 마그마는 화산 폭발과 흘러내리는 용암, 전기는 여러 번 강하게 내려치는 번개로 화면을 채우고 획득 문구 제거
- 수집 조각을 최초 크기의 50%로 줄이되 체감 터치 범위 유지
- 서로 다른 특수 연출·소리는 겹쳐 재생하고 같은 종류가 다시 나올 때만 이전 같은 종류를 교체
- 전기 조각의 저급한 지지직음을 강한 근거리 낙뢰음으로 교체
- 강화 화면에 보이는 쿠키 관련 7종 전체를 메인 쿠키 진화 레벨과 다음 진화까지 남은 강화 횟수에 소급 반영
- 고레벨 비용 증가에 비해 약했던 클릭 힘의 명시 레벨 이후 증가량을 +25에서 +50으로 상향
- 10% 확률이 9번 연속 실패하면 10번째에 확정되는 규칙을 4종 희귀 보상에 적용하고 앱 종료 후에도 실패 횟수 유지
- 낮은 확률일수록 연출·음향이 웅장하게 전기 > 슈퍼 > 마그마 > 일반 순서를 보장하고 슈퍼가 마그마보다 밋밋한 문제 개선

### 구현

- 네 희귀 보상의 확률 눈금을 200,000으로 통일했다. 모든 강화는 레벨마다 기본 확률의 25%, 기본 보상 배수의 10%만큼 증가한다. 일반 `1%·10배 → +0.25%p·+1배`, 슈퍼 `0.1%·100배 → +0.025%p·+10배`, 마그마 `0.2%·50배 → +0.05%p·+5배`, 전기 `0.05%·200배 → +0.0125%p·+20배`다.
- 네 비율의 교차 곱과 명시 레벨 값을 앱 시작 검증에서 강제한다. 마그마·전기 확률은 5%·1.5% 상한에서 멈추지만 `maximumLevel`은 없으며 보상 배수·가격·단계는 계속 증가한다. 기존 저장은 보유 레벨에 새 배수 공식을 자동 소급한다.
- 슈퍼·두 조각의 `countsTowardCookieEvolution`을 `true`로 맞췄다. 강화 화면에 보이는 7종 전체가 기여하고 새 게임 기본 합계는 7이며 기존 슈퍼·조각 레벨도 메인 진화 진행률에 즉시 포함된다.
- 클릭 힘은 Lv.8의 55 이후 레벨당 +50, 비용 ×1.25로 조정했다. 자동 생산의 +25보다 정확히 두 배 높은 순간 성장이고, 크리티컬·조각은 최종 클릭 힘을 사용하므로 함께 강화된다.
- 수집 조각은 112px에서 정확히 50%인 56px로 줄이고 hit slop은 40px로 넓혀 전체 터치 폭을 기존과 거의 같게 유지했다. 좌우 교대 발사, 9개 부스러기, 맥동 오라와 5초 남은 시간 막대는 유지한다.
- 마그마 획득용 초콜릿 쿠키 화산을 새로 생성·투명 WebP 변환했다. 화면보다 크게 확장한 화산 폭발에 11줄기 하강 용암과 22개 불씨를 합성한다. 전기는 7갈래×9분절 낙뢰와 청백색 파편 20개를 사용한다. 별도 획득 문구는 제거하고 실제 증가량·배수만 표시한다.
- `CookieSpecialFeedback`은 종류별 항목을 최대 4개 유지한다. 새 항목은 같은 종류만 필터링하고 다른 종류는 계속 표시한다. 일반 클릭 오디오와 특수 오디오를 별도 훅으로 분리했으며, 특수 사운드도 종류별 플레이어·예약 타이머·요청 세대를 갖는다.
- 전기 조각은 Mixkit `Strong close thunder explosion`을 최대 상대 음량 1.0으로 220ms 간격 세 번 재생한다. 이전 지지직·전자 폭발 파일은 런타임과 저장소에서 제거했고 새 파일의 공식 페이지·직접 URL·SHA-256을 기록했다.
- 시각 수명을 일반 930ms < 마그마 1,500ms < 슈퍼 1,800ms < 전기 1,950ms로 정렬했다. 슈퍼에 화면 상단 천공 균열, 5중 참격, 14갈래×5분절 번개와 24개 파편을 추가했고, 마그마 화염은 180ms 간격 두 겹으로 놓였다. 시각 수명·음향 레이어 강도·기본 확률 역순을 교차 테이블 검증으로 강제한다.
- `cookie-pity.json`을 추가하고 현재 확률의 역수를 최대 시도 횟수로 사용한다. 자연 슈퍼는 일반 천장으로 낮아지지 않고, 전기·마그마 동시 확정은 전기를 먼저 주되 마그마 횟수를 유지한다.
- 저장 스키마를 v13, 앱을 1.0.17(18)로 올렸다. `cookiePityMisses` 4종을 클릭 보상과 한 리듀서 전이에서 저장하고 이전 저장은 0, 손상값은 안전 정수로 정규화한다.

## 2026-07-19 · 비원형 크리티컬 연출과 슈퍼 확률 소급

### 요청

- 원형으로 퍼지는 일반·슈퍼 크리티컬 이펙트를 더 강하고 게임다운 연출로 교체
- 슈퍼 크리티컬에 `콰광` 같은 고품질 폭발음 적용
- 슈퍼 크리티컬 확률을 레벨마다 +0.025%p 올리고 현재 강화 레벨에 소급

### 구현

- 일반 크리티컬의 원형 코어·이중 링을 제거하고 교차 참격광, 6갈래 번개 균열, 각진 쿠키 파편으로 교체했다.
- 슈퍼 크리티컬의 3색 링·방사 원형 구성을 제거하고 수직 낙뢰, X자·수평 다색 참격과 잔상, 10갈래 번개, 16개 파편 폭발, 쿠키 무대 흔들림으로 교체했다.
- 공통 섬광·참격은 `AngularImpactPrimitives`, 번개·파편은 `AngularBurstParticles`로 분리하고 개수·크기·색상·키프레임은 `cookie-feedback.json`에서 관리한다. 애니메이션은 native driver를 사용한다.
- Mixkit `Heavy electric shockwave impact`를 내려받아 기존 시네마틱 충격 110ms 뒤에 겹쳤다. 원본 URL, 라이선스, 6.69초 인코딩 정보와 SHA-256을 기록했다.
- 두 크리티컬 확률 눈금을 100,000단위로 높였다. 일반 확률은 동일하게 유지하고 슈퍼는 Lv.1 0.1%에서 레벨당 정확히 +0.025%p가 되도록 명시·무한 레벨 테이블을 함께 변경했다.
- 저장된 강화 레벨로 현재 확률을 계산하므로 기존 진행도는 별도 저장 이전 없이 자동 소급된다. Lv.20 회귀 테스트는 0.575%를 확인한다.

## 2026-07-19 · 슈퍼 크리티컬, 자동 전투, 클릭 입력 개선

### 요청

- 기본 0.1%·100배의 쿠키 슈퍼 크리티컬과 전용 고품질 연출·사운드
- 성 자동 공격과 승리 후 다음 스테이지 연속 진행을 제공하는 저장형 자동 전투 모드
- 빠른 연속 터치와 손가락 이동에도 잘 반응하는 메인 쿠키 입력

### 구현

- 슈퍼 크리티컬을 일반 크리티컬보다 먼저 판정하는 배타적 단일 난수 구간으로 추가하고, 확률 10% 상한 뒤에도 배수가 계속 성장하는 무한 강화를 추가했다.
- 수치·가격·확률·사운드·시각 효과는 `cookie-super-critical.json`, `cookie-upgrades.json`, `cookie-upgrade-rules.json`, `cookie-feedback.json`으로 분리했다.
- Mixkit의 `Movie trailer epic impact`와 `Choir magic shine`을 전용 MP3로 번들하고 출처·라이선스·SHA-256을 기록했다.
- 슈퍼 전용 섬광·3색 링·방사 광선·별빛 컴포넌트와 전체/축약 피드백 정책을 추가했다.
- `battle-auto.json`과 저장 상태 `autoBattleEnabled`를 추가했다. 자동 모드는 전투 시작, 기존 엔진 명령을 통한 성 자동 공격, 승리 보상 확정 뒤 다음 스테이지 진행을 수행한다. 패배와 최종 난이도 마지막 전투는 자동 연결하지 않는다.
- `cookie-input.json`의 hit slop·이동 허용 범위·중복 제거 시간을 사용해 터치 시작 즉시 보상하고 같은 제스처의 후속 `onPress`를 제거했다. 접근성의 `onPress` 전용 입력은 유지했다.
- 저장 스키마를 v11, 앱을 1.0.14(15)로 올리고 이전 저장은 슈퍼 크리티컬 Lv.1·자동 전투 OFF로 안전하게 복구한다.
- 메인 화면 피드백 렌더링을 `CookieGainFeedback`로 분리해 `GameScreen`을 178줄로 유지했다.
