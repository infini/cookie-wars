## 2026-07-19 · A·B 순차 대결·클릭 힘 상향·축약 수치·희귀 연출 개선·1.0.24 설치

### 요청

- 두 사람이 동시에 누르지 않고 A가 정한 시간 동안 먼저 클릭한 뒤 B가 같은 시간 동안 이어서 도전하고, 마지막에 두 기록과 승자를 보여주는 미니게임 추가
- 제한 시간은 10초 단위로 설정하고 하단 메뉴에서 한 번에 진입
- 클릭 힘을 Lv.1 100, 강화마다 +100으로 높이고 기존 강화 단계에도 소급 적용
- 일반·슈퍼 크리티컬은 예전 선형 연출로 되돌리고, 현재 전기 스타일은 유지하면서 번개를 세 줄로 확대하며 마그마는 고품질 외부 분출 애니메이션으로 교체
- 원반 공격력을 비롯한 긴 게임 수치를 천·만·억·조 축약형으로 표시
- 문서·릴리스 APK·실기기 설치·Git 푸시까지 완료

### 구현

- `MiniGameScreen`을 추가해 10~60초를 10초 단위로 선택하고 A 카운트다운·플레이·폰 인계·B 카운트다운·플레이·결과 순서로 진행한다. 대결 점수는 메인 쿠키 저장과 완전히 분리했고 같은 시간 재대결과 시간 재설정을 제공한다.
- 미니게임 시간 범위·기본값·카운트다운·입력 애니메이션은 `mini-game.json`, 시간 조정·플레이어 전이·승자 판정은 `domain/miniGame.ts`, 실제 종료 시각과 클릭 허용 경계는 `useMiniGamePhaseTimer`로 분리했다. 하단은 게임·대결·전투·강화·정보 5개 대분류다.
- 클릭 힘 테이블의 명시 Lv.1~8을 100~800으로 바꾸고 무한 강화 증가량도 +100으로 변경했다. 저장은 레벨만 보유하므로 기존 사용자는 저장 초기화 없이 현재 단계에 새 공식을 즉시 적용받는다.
- 공통 `formatNumber`가 긴 정수를 한 자리 소수의 천·만·억·조로 축약한다. 원반 현재/다음 공격력·강화 비용·레벨뿐 아니라 쿠키·자동 생산·HP·피해량·봇 수량·전투 보상·대결 점수까지 같은 함수를 사용하고, 확률·시간·소수 배율은 정확한 전용 표기를 유지한다.
- 일반·슈퍼는 `CookieLineBurstEffect`의 공통 native-driver 선형 섬광으로 복원하고 선 개수·길이·색·진행률은 `cookie-special-effects.json`으로 이동했다. 슈퍼는 일반보다 선이 길고 조밀하며 금색·청록·보라 보조선을 사용한다.

## 2026-07-19 · 외부 16/64프레임 희귀 보상 연출·1.0.22 실기기 설치

### 요청

- 일반 크리티컬 연출을 다시 표시하고 `크리티컬 < 마그마 < 슈퍼 크리티컬 < 전기` 순으로 연출 품질·시간·화면 점유 범위를 키우기
- 직접 그린 단순 도형 대신 무료 사용 가능한 고품질 외부 애니메이션 원본 사용
- 기존 기준에서 마그마 10%, 슈퍼 크리티컬 20%, 전기 30% 확대
- 크리티컬은 중앙, 마그마는 중앙 왼쪽 위, 슈퍼 크리티컬은 중앙 오른쪽 위, 전기는 중앙 아래에 배치
- 획득 쿠키 숫자 크기를 30% 확대하고 연결 기기 설치·릴리스 APK·문서·Git 푸시까지 완료

### 구현

- OpenGameArt에서 CC0로 제공되는 Sinestesia의 16프레임 노란 타격과 64프레임 폭발, Esteban Díaz/Inguz Media의 64프레임 금색·청록 충격 atlas를 사용했다. 원본 주소·선택 라이선스·SHA-256은 `assets/licenses/OPENGAMEART_COOKIE_VFX_SOURCE.txt`에 기록했다.
- `process_external_cookie_feedback_vfx.py`가 모든 프레임의 합집합 알파 경계를 한 번 계산해 흔들림 없이 투명 여백을 자른다. 크리티컬은 노란 타격, 마그마는 주황 화염 폭발, 슈퍼는 금색·청록 마법 충격 합성, 전기는 3회에 걸쳐 9개 낙뢰와 청록 충격을 합성한 애니메이션 WebP로 변환한다.
- `withAnimatedWebp` Expo config plugin으로 Android Fresco의 animated WebP 지원을 prebuild마다 활성화했다. 네 연출은 프레임별 React 상태·JS 타이머 없이 네이티브 이미지 디코더에서 재생되며 발생한 동안에만 마운트된다.
- `cookie-special-effects.json`을 추가해 네 종류의 전체·축약 수명, 최소 크기, 화면 비율, 위치 offset, z-index, 원본 프레임 수와 조각 보상 글자 표현을 한 테이블로 분리했다. 시작 검증은 네 ID의 누락·중복과 희귀도에 따른 시간·크기·점유율·레이어 순서 역전을 거부한다.
- 같은 종류가 다시 발동하면 그 종류의 시각·음향만 교체하고 다른 종류는 합성하는 기존 규칙을 유지했다. 일반 크리티컬도 중앙에 복원했으며 마그마·슈퍼·전기는 지정 위치와 110%·120%·130% 확대값을 적용했다.
- 일반·크리티컬·슈퍼 획득 숫자와 마그마·전기 수령 숫자를 직전 값의 정확히 130%로 확대했다. 이전 절차적 도형 컴포넌트 9개와 거대한 중복 설정·검증 모듈을 제거하고 공통 네이티브 WebP 재생기로 단순화했다.
