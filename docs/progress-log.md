# 작업 진행 기록

## 2026-07-19 · 메인 UI·희귀 연출 동작 개선·Black Sun 15개 난이도

### 요청

- 메인 상단은 `쿠키전쟁`과 한 줄 현재 쿠키 이름, 보유 쿠키·진화 레벨과 다음 진화 진행 막대만 표시
- 전투 이동 버튼과 안내 문구를 제거하고 중앙 쿠키를 확대하며 하단은 전투 훈장·자동 생산·클릭 획득·두 크리티컬·두 조각을 간결하게 정리
- 일반 크리티컬은 숫자만 표시하고 마그마·슈퍼·전기는 고정 이미지가 아닌 시간차 동작과 충격을 갖춘 연출로 개선
- 화산과 화염이 따로 보이지 않도록 분화구와 수직 화염의 기준점을 일치
- 기존 30개 뒤에 15개 난이도를 더하고 이름은 `black sun easy`부터 `black sun extreme god`까지 사용
- 기존 빌드를 먼저 휴대폰에 설치한 뒤 연출 개선 최종 빌드도 다시 설치

### 구현

- `TopBar`에 게임 화면 전용 표현을 분리했다. 게임에서는 부제와 쿠키 잔액 pill 대신 넓은 한 줄 현재 쿠키 이름을 표시하고, 다른 화면의 제목·잔액은 유지한다.
- `GameScreen`은 보유 쿠키·진화 레벨, 다음 진화까지 남은 강화 수와 진행 막대, 확대 쿠키, 전투 훈장→자동 생산→클릭 획득 순의 하단 정보만 조합한다. 일반/슈퍼와 마그마/전기는 프로젝트 전용 아이콘 두 줄로 묶었다. 쿠키 렌더 상한은 JSON의 280px로 확대했다.
- 획득 숫자는 기존의 50% 수준으로 줄이고 `일반 → 크리티컬 → 마그마 → 슈퍼 → 전기` 순으로 색·굵기를 구분했다. 일반 크리티컬의 화면 이펙트는 제거하되 전용 소리와 노란 숫자는 유지했다.
- 마그마는 분화구 중심에 맞춘 프로젝트 전용 화염 기둥이 4회 맥동·좌우 흔들리고, 세 겹 타원 충격파와 16개 불씨가 올라오도록 구성했다. 전기는 외부 CC0 낙뢰 11개가 시차 점멸하고 세 겹 마름모 충격파·18개 방사 파편·회전 코어를 합성한다. 슈퍼 크리티컬은 충전 마름모, 3회 충격파, 순차 참격, 8갈래 번개와 회전 엠블럼을 사용한다. 모두 native-driver 단일 진행값을 사용하며 수치와 순서는 JSON에서 검증한다.
- `difficulties.json`과 적 원반을 `black sun easy`~`black sun extreme god` 15개로 확장해 총 45개로 만들었다. 새 난이도의 첫 전투 HP·공격력과 적 원반 피해는 매번 직전 난이도 20번째보다 20% 높다. Black Sun은 이미 제작한 기본 15개 세계·보스를 더 강한 재대결로 사용해 런타임 전투 개체 수를 늘리지 않는다.
- 저장 스키마를 v15로 올리고 완주한 캠페인 수에 따라 v14의 30개 완료 저장은 `black sun easy`로 한 번 연결한다. 난이도 확장 이전을 배열 데이터로 바꿔 후속 시리즈도 같은 구조로 추가할 수 있게 했다.
- 앱을 1.0.21(22)로 올리고 `artifacts/cookie-wars-v1.0.21.apk`를 생성한다.

### 검증

- `npm run verify`: TypeScript, Jest 16개 스위트·261개 테스트, Expo Doctor 20/20 통과
- Android release: Metro 1,312개 모듈·397개 에셋, APK 127,178,685바이트, SHA-256 `2494cd92afdb458008c616f0c277ebd9e737c9475e2c5edd1e3e80ccfca35302`
- APK: `com.cookiewars.game`, versionCode 22, versionName 1.0.21, API 24~36, 4개 ABI, v2 서명을 최종 검증
- 실기기 강제 QA에서는 마그마·슈퍼·전기의 여러 진행 프레임을 캡처해 중심 정렬과 시간차 동작을 확인했다. 강제 발동 코드는 최종 소스에서 제거했다.
- 최대 프로덕션 TypeScript는 `GameScreen.tsx` 296줄, `gameSave.ts` 293줄이며 300줄을 넘는 실행 파일이 없다.
- Xiaomi `24069PC21G`에 1.0.20(21) 위로 1.0.21(22)을 덮어 설치했다. 최초 설치 시각과 저장을 유지했고 콜드 실행 298ms, `MainActivity` resumed, 최근 로그에서 AndroidRuntime·React Native 치명 오류와 ANR이 없음을 확인했다.

### 알려진 한계와 후속

- Black Sun은 새 배경 색 변형이 아니라 기본 15개 세계·보스의 의도적인 고난도 재대결이다. 신규 시각 테마가 필요하면 별도 에셋 제작 작업으로 추가한다.
- 희귀 연출은 프레임별 React 상태를 만들지 않았지만 장시간 플레이의 FPS·발열은 사용자 세션에서 계속 체감 점검한다.
- 현재 APK는 직접 테스트용 개발 서명이다. 공개 스토어 배포 전에는 production signing key가 필요하다.

## 2026-07-19 · 전투 중 절전 방지·80종 쿠키·클릭 성장·외부 VFX

### 요청

- 실제 전투 중에는 화면 자동 절전을 막고, 전투 종료·결과·다른 메뉴에서는 시스템 절전을 허용
- 현재 최종 진화 뒤에 쿠키 30종을 추가하고 클릭 힘을 기본 50, 강화마다 +50으로 조정
- 강화 가능 항목을 위에 두고 그중 강화 비용이 저렴한 항목을 먼저 표시
- 마그마는 화산에서 분출되고 전기는 여러 번 내려치되 화면 전체를 가리지 않는 외부 고품질 연출 사용, 기존 사운드는 유지
- 아들 폰에도 옮겨 설치할 수 있는 릴리스 APK 생성, 연결 기기 설치와 Git 푸시

### 구현

- `expo-keep-awake` 네이티브 모듈과 `useBattleKeepAwake`를 추가했다. 전투 상태가 정확히 `active`일 때만 태그 기반 Wake Lock을 획득하고 idle·victory·defeat 전환과 화면 unmount에서 같은 태그를 즉시 해제한다. 순수 정책 함수와 회귀 테스트로 다른 메뉴 절전 허용을 고정했다.
- 쿠키를 50종에서 80종으로 확장했다. 51단계 수정 용의 알은 진화 Lv.375, 이후 12레벨 간격으로 80단계 쿠키 우주핵 Lv.723까지 이어진다. 공통 능력 배율은 신규 단계마다 직전 값의 ×1.10을 소수 둘째 자리로 반올림하며 최종 ×1083.16이다.
- `cookie-expansion.json`과 `validateCookieExpansion`을 추가해 기존·확장 개수, 첫 요구 레벨, 간격과 배율을 데이터로 분리하고 모든 신규 행을 앱 시작 시 역산한다. 30종은 8개 생성 atlas에서 고유 실루엣으로 제작하고 90% 중앙 여백·크로마키·알파 검증을 거쳐 512×512 투명 WebP로 등록했다.
- 클릭 힘은 Lv.1 50, Lv.2~8은 100~400, 이후도 강화마다 +50으로 통일했다. 저장은 레벨 기반이므로 기존 단계에 새 값이 자동 적용된다.
- 강화 selector는 `구매 가능 → 쿠키 부족 → 완료` 그룹을 유지하면서 같은 그룹 안에서 다음 비용 오름차순, 동률은 테이블 순서로 안정 정렬한다.
- 마그마의 11줄기 하강 용암과 전기의 절차적 분절을 제거했다. OpenGameArt CC0 폭발 atlas 16프레임을 화산 분화구 위에서 한 번 재생하고, CC0 번개 11종 중 7종을 서로 다른 위치와 시차로 표시한다. 화면 점유율과 플래시를 낮췄고 요청대로 음원·음량 설정은 변경하지 않았다. 원본 주소·라이선스·SHA-256과 재현 변환 스크립트를 기록했다.
- 앱을 1.0.19(20)로 올리고 아들 폰에 전달 가능한 `artifacts/cookie-wars-v1.0.19.apk`를 생성했다.
- 100MB를 넘는 최종 APK는 Git LFS로 추적해 저장소에서 직접 내려받을 수 있게 했다.

- 이미지 검증: 쿠키 WebP 80개 모두 512×512 RGBA와 알파 `(0, 255)`, 외부 VFX WebP 27개 RGBA와 알파 `(0, 255)`, 모든 정적 레지스트리 연결 통과

## 2026-07-19 · Blood Moon 15개 난이도·전투 훈장 상세 안내

### 요청

- 기존 난이도 수만큼 신규 난이도를 추가하고 이름은 기존 이름 앞에 `blood moon`을 붙이기
- 신규 구간은 난이도가 넘어갈 때마다 직전 난이도보다 추가로 20% 어려워지게 구성
- 전투 훈장의 `쿠키 성장 +n%`가 실제로 무엇을 강화하는지 `정보` 화면에서 상세히 설명
- 기존 15개 난이도를 모두 완료한 진행도를 신규 캠페인에 자동 연결
- 신규 난이도·맵·보스 작업 뒤 코드 복잡도와 성능 경계를 검토하고 APK 설치·Git 푸시까지 완료

### 구현

- 기존 15개 뒤에 `blood moon easy`부터 `blood moon extreme god`까지 15개를 추가해 총 30단계로 확장했다. 신규 각 단계의 1번째 보스 HP·공격력은 직전 단계 20번째보다 정확히 20% 높고 적 원반 피해도 단계마다 20% 증가한다.
- `difficulty-expansion.json`에 기존/신규 개수, +20% 성장률, 이동 속도와 적 원반 크기·속도·쿨타임 상하한을 분리했다. 전용 검증기가 모든 신규 경계값과 원반 레벨을 역산해 잘못된 데이터가 실행되지 않게 한다.
- 신규 15개 난이도마다 재색칠이 아닌 수정 광산·버섯 숲·증기 주조소·심해 산호 도시·운석 분화구·창조 공방·무한 공간 계열의 독립 전장과 고유 망치 보스를 제작했다. 런타임 구성은 전장·정적 보스 각 30개, 보스 애니메이션 180프레임이다.
- 저장 스키마를 v14로 올렸다. v13 이하에서 기존 15개를 모두 20승 완료한 저장은 과거 난이도 선택 위치와 무관하게 최초 실행 한 번만 `blood moon easy`를 해금·선택하며, 기존 300승·300훈장·보상 기록을 보존한다.
- `정보 > 쿠키·훈장`에 현재 훈장 수, 클릭 힘·자동 생산·쿠키 성 체력의 실제 +퍼센트와 배수, 훈장당 +1%, 적용 계산식, 다음 획득 전투와 재도전 무보상 규칙을 추가했다.
- 신규 성장 검증과 저장 선택 이전을 독립 모듈로 분리했다. 최대 프로덕션 TypeScript 파일은 298줄이고 실행 파일은 모두 300줄 이하를 유지한다.

## 2026-07-19 · 무한 쿠키 조각 성장·합성 피드백·강한 낙뢰음

### 요청

- 쿠키 클릭 때 0.2% 마그마·0.05% 전기 조각을 옆으로 튀어나오게 하고 5초 안에 누르면 Lv.1 기준 클릭 힘 50배·200배 지급
- 조각 강화도 크리티컬처럼 발견 확률과 획득 배수가 함께 무한 성장하고, 네 희귀 보상의 성장 비율을 동일하게 구성

- 수집 조각을 최초 크기의 50%로 줄이되 체감 터치 범위 유지
- 서로 다른 특수 연출·소리는 겹쳐 재생하고 같은 종류가 다시 나올 때만 이전 같은 종류를 교체

- 강화 화면에 보이는 쿠키 관련 7종 전체를 메인 쿠키 진화 레벨과 다음 진화까지 남은 강화 횟수에 소급 반영
- 고레벨 비용 증가에 비해 약했던 클릭 힘의 명시 레벨 이후 증가량을 +25에서 +50으로 상향

- 낮은 확률일수록 연출·음향이 웅장하게 전기 > 슈퍼 > 마그마 > 일반 순서를 보장하고 슈퍼가 마그마보다 밋밋한 문제 개선

### 구현

- 네 희귀 보상의 확률 눈금을 200,000으로 통일했다. 모든 강화는 레벨마다 기본 확률의 25%, 기본 보상 배수의 10%만큼 증가한다. 일반 `1%·10배 → +0.25%p·+1배`, 슈퍼 `0.1%·100배 → +0.025%p·+10배`, 마그마 `0.2%·50배 → +0.05%p·+5배`, 전기 `0.05%·200배 → +0.0125%p·+20배`다.
- 네 비율의 교차 곱과 명시 레벨 값을 앱 시작 검증에서 강제한다. 마그마·전기 확률은 5%·1.5% 상한에서 멈추지만 `maximumLevel`은 없으며 보상 배수·가격·단계는 계속 증가한다. 기존 저장은 보유 레벨에 새 배수 공식을 자동 소급한다.
- 슈퍼·두 조각의 `countsTowardCookieEvolution`을 `true`로 맞췄다. 강화 화면에 보이는 7종 전체가 기여하고 새 게임 기본 합계는 7이며 기존 슈퍼·조각 레벨도 메인 진화 진행률에 즉시 포함된다.
- 클릭 힘은 Lv.8의 55 이후 레벨당 +50, 비용 ×1.25로 조정했다. 자동 생산의 +25보다 정확히 두 배 높은 순간 성장이고, 크리티컬·조각은 최종 클릭 힘을 사용하므로 함께 강화된다.
- 수집 조각은 112px에서 정확히 50%인 56px로 줄이고 hit slop은 40px로 넓혀 전체 터치 폭을 기존과 거의 같게 유지했다. 좌우 교대 발사, 9개 부스러기, 맥동 오라와 5초 남은 시간 막대는 유지한다.
- 마그마 획득용 초콜릿 쿠키 화산을 새로 생성·투명 WebP 변환했다. 화면보다 크게 확장한 화산 폭발에 11줄기 하강 용암과 22개 불씨를 합성한다. 전기는 7갈래×9분절 낙뢰와 청백색 파편 20개를 사용한다. 별도 획득 문구는 제거하고 실제 증가량·배수만 표시한다.
- `CookieSpecialFeedback`은 종류별 항목을 최대 4개 유지한다. 새 항목은 같은 종류만 필터링하고 다른 종류는 계속 표시한다. 일반 클릭 오디오와 특수 오디오를 별도 훅으로 분리했으며, 특수 사운드도 종류별 플레이어·예약 타이머·요청 세대를 갖는다.
- 전기 조각은 Mixkit `Strong close thunder explosion`을 최대 상대 음량 1.0으로 220ms 간격 세 번 재생한다. 이전 지지직·전자 폭발 파일은 런타임과 저장소에서 제거했고 새 파일의 공식 페이지·직접 URL·SHA-256을 기록했다.
- 시각 수명을 일반 930ms < 마그마 1,500ms < 슈퍼 1,800ms < 전기 1,950ms로 정렬했다. 슈퍼에 화면 상단 천공 균열, 5중 참격, 14갈래×5분절 번개와 24개 파편을 추가했고, 마그마 화염은 180ms 간격 두 겹으로 놓였다. 시각 수명·음향 레이어 강도·기본 확률 역순을 교차 테이블 검증으로 강제한다.
- `cookie-pity.json`을 추가하고 현재 확률의 역수를 최대 시도 횟수로 사용한다. 자연 슈퍼는 일반 천장으로 낮아지지 않고, 전기·마그마 동시 확정은 전기를 먼저 주되 마그마 횟수를 유지한다.
