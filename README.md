# 쿠키전쟁 (Cookie Wars)

아이가 기획한 손그림을 바탕으로 만든 Android 전용 세로형 싱글 플레이 게임입니다. 쿠키를 모으는 화면과 전투 화면을 완전히 분리했으며, 화면 아래의 게임·대결·전투·강화·정보 5개 대분류와 간결한 소메뉴로 이동합니다.

## 설치

- APK: [`artifacts/cookie-wars-v1.0.25.apk`](artifacts/cookie-wars-v1.0.25.apk)
- APK 크기: 139,174,192바이트 (132.73MiB)
- SHA-256: `7f83bd413555ca4168c51ecfbedd303b1b449bc8d770ac6f1045d4e9a53b51b3`
- 패키지명: `com.cookiewars.game`
- 지원 범위: Android 7.0(API 24) 이상
- 화면 방향: 세로 고정

APK를 Android 스마트폰으로 옮긴 뒤 파일을 열어 설치합니다. Play 스토어가 아닌 APK를 직접 설치하므로, 스마트폰에서 해당 파일 관리 앱의 **알 수 없는 앱 설치 허용**이 필요할 수 있습니다.

릴리스 APK는 GitHub의 일반 파일 크기 제한을 넘으므로 Git LFS로 관리합니다. 저장소를 새로 받은 뒤 APK가 포인터 파일로 보인다면 `git lfs install`과 `git lfs pull`을 실행합니다.

현재 APK는 직접 테스트·설치를 위한 개발 서명 빌드입니다. Play 스토어 배포 전에는 별도의 안전한 배포 키로 서명해야 합니다.

## 현재 구현

- 가운데 전체 쿠키 클릭 영역, 클릭 애니메이션, 획득 텍스트, 효과음·진동
- 10초 단위로 시간을 고른 뒤 A가 먼저, B가 이어서 같은 쿠키를 누르고 두 기록과 승자를 보여주는 로컬 2인 순차 클릭 대결
- 메인 상단의 한 줄 현재 쿠키 이름과 보유량·진화 레벨·다음 진화 진행 막대, 280px까지 확대한 중앙 쿠키, 하단 전투 훈장·자동 생산·클릭 획득과 희귀 보상 아이콘
- 화면에 표시되는 데이터 기반 강화 7종 모두 무한 성장, 네 희귀 보상의 확률 상한 뒤에도 배수 성장 지속, 기존 크기 레벨의 1회 저장 이전
- 별도 경험치 없이 진화 기여 강화 레벨과 기존 저장 보너스의 합계로 자동 진화하는 80종 쿠키
- 기본 확률 1%·획득량 10배의 쿠키 크리티컬과 확률 50% 상한·배수 무제한의 무한 강화
- 기본 확률 0.1%·획득량 100배, 강화당 확률 +0.025%p가 기존 레벨에도 소급되는 배타적 쿠키 슈퍼 크리티컬과 확률 10% 상한·배수 무제한 강화
- 클릭 시 마그마 0.2%·전기 0.05% 확률로 옆으로 튀어나와 등장하고 5초 안에 직접 누르면 기본 클릭 힘의 50배·200배를 받는 쿠키 조각
- 마그마·전기 조각은 강화마다 기본 대비 발견 확률 +25%, 획득 배수 +10%가 함께 오르고 확률 상한 뒤에도 배수가 계속 성장
- 표시 확률의 역수만큼 연속 실패하면 다음 클릭에서 확정 발동하는 4종 독립 천장 보정과 재실행해도 유지되는 실패 횟수
- 강화 화면에 보이는 쿠키 관련 7종 전체의 현재 단계가 쿠키 진화 레벨과 다음 쿠키까지 남은 강화 횟수에 즉시 소급 반영
- 5초 타이머, 좌우 교대 발사·부스러기 낙하와 최초 크기의 50%로 줄인 전용 투명 WebP 수집 에셋·동일한 체감 터치 범위
- Freesound CC0 `Crunch`를 세 가지 속도로 번갈아 쓰는 쿠키 클릭음과 Mixkit 충격·반짝임을 겹친 크리티컬 사운드
- CodeManu CC0 원본을 사용한 60프레임 네온 옐로 타격 크리티컬·형광 오렌지와 핑크 외곽광의 마그마·네온 마젠타와 시안 소용돌이 슈퍼 크리티컬, 형광 시안과 보라 외곽광의 세 줄 동시 낙뢰 전기 연출
- 네 희귀 연출을 모두 중앙에 배치하고 색·실루엣·지속 시간·화면 점유 범위로 종류와 희귀도를 구분
- `크리티컬 < 마그마 < 슈퍼 크리티컬 < 전기` 순으로 지속 시간·화면 점유 범위·레이어 밀도를 높이고 마그마 +10%·슈퍼 +20%·전기 +30% 크기 확대
- 네 종류는 서로 겹쳐 합성하되 같은 종류가 다시 나오면 이전 같은 종류의 시각·음향만 새 연출로 교체
- 모든 획득 숫자를 기존 대비 30% 키우고 시각·사운드는 희귀도가 높을수록 강해지도록 교차 테이블 검증
- 전기 조각의 지지직음을 제거하고 Mixkit `Strong close thunder explosion`을 220ms 간격으로 세 번 겹친 강한 낙뢰음
- 터치 시작 즉시 처리, 넓은 hit slop·이동 허용 범위와 동일 제스처 중복 방지로 개선한 쿠키 클릭 입력
- 정보 화면에서 확인하는 전체 진화 순서·다음 쿠키 필요 레벨·남은 강화 수
- 길 없이 넓은 중앙 공간을 유지하는 30개 고유 전장과 기본 15개 세계에서 재대결하는 Black Sun 캠페인
- 넓어진 공격 반경 안의 적만 노리는 쿠키봇·쿠키 성 원반
- 성 앞에서 멈춘 적도 원반 속도와 관계없이 반드시 명중하는 추적 충돌
- 느린 비행·회전으로 궤적이 보이는 원반과 작은 원근 유닛
- 파랑·금색 아군 원반과 빨강·검붉은색 적 원반
- 적 수는 항상 한 마리이며 보스 능력치로 조절되는 난이도
- 아군 이름 파란색, 적군 이름 빨간색 표시
- 저렴하게 구매 가능하고 등급별 능력 격차가 큰 쿠키 원반 5종과 쿠키봇 5종
- 강화 아래에서 쿠키·원반·쿠키봇 강화를 바로 고르는 3개 소메뉴
- 정보 아래에서 쿠키·훈장·난이도·몬스터를 바로 고르는 3개 소메뉴와 NEW 배지
- 상위 종류로 갈수록 큰 폭으로 벌어지는 원반·쿠키봇 공격 능력
- 종류별 영구 소유·장착과 제한 없는 원반 강화
- 각 봇 위치에서 나오는 작은 자동 원반과 전투 화면 어디든 누르면 성에서 나오는 큰 2배 원반

- 전투 승리로 쿠키를 직접 지급하지 않고, 전투 훈장이 쿠키 획득·생산·성 체력을 영구 강화
- 몬스터 도감의 `NEW` 및 잠금 표시

- 저장 스키마 v15와 기존 15개 완료 저장의 `blood moon easy`, 기존 30개 완료 저장의 `black sun easy` 자동 연결, 전투 속도·자동 전투·두 크리티컬·두 조각 강화·4종 천장 실패 횟수의 안전한 복구

- 모든 사용자 화면을 한국어로 구성

## 개발과 빌드

Node.js와 Android SDK/JDK가 설치된 환경에서 실행합니다.

```bash
npm install
npm run verify
npm run prebuild
npm run android:release
```

빌드 결과는 `android/app/build/outputs/apk/release/app-release.apk`에 생성됩니다. 실제 기기에서 개발 모드로 실행하려면 USB 디버깅을 켜고 `npm run android:device`를 사용합니다.

## 구조

```text
src/
├── components/   공통 모바일 UI
├── config/       난이도·몬스터·업그레이드·무기 JSON 데이터
├── domain/       성장·진행·보상 순수 계산
├── engine/       프레임 기반 전투 로직
├── navigation/   5대 메뉴·소메뉴 데이터와 검증 모델
├── screens/      9개 leaf 화면과 전투·미니게임 전용 모듈
├── services/     저장, 효과음, 진동
├── state/        영구 상태·저장 이전·명령·자동 생산

└── types/        전투·성장·저장·UI 영역별 TypeScript 데이터 계약
```

## 주요 명령

- `npm start`: Expo 개발 서버
